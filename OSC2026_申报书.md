## 二、项目简介

本项目是一个以 MoonBit 为主要实现语言的系统型基础库，面向状态驱动的业务逻辑、AI Agent 流水线、游戏行为树、协议状态管理和复杂前端交互等场景，提供三类核心能力：

- 分层有限状态机（HSM）：支持父子状态、事件冒泡、跨层级跳转，以及基于最近公共祖先（LCA）的生命周期执行路径。
- 工作流编排（Workflow）：支持顺序化步骤执行、失败重试、状态追踪和失败回调，适合 Agent 执行管线与批处理任务。
- 确定性定时器（Timer）：支持可测试、可复现的超时与周期触发模拟，避免真实线程与睡眠带来的不稳定性。

同时，项目提供 Mermaid / PlantUML 导出能力，便于把运行结构同步到文档、设计稿和 README 中。

## 三、项目目标与应用场景

### 1. 解决的问题

复杂软件系统中，状态变化、任务推进和定时触发经常交织在一起，常见痛点包括：

- 状态层级难以表达，父状态逻辑复用不方便
- 多步骤任务缺少统一的失败恢复和重试机制
- 定时逻辑依赖真实时间，测试不稳定
- 架构图和实现代码分离，文档容易过期

### 2. 适用场景

- AI Agent 自动执行流水线
- 游戏 NPC / Boss 行为状态机
- 网络协议握手与超时管理
- 前端复杂多步骤表单与权限流转
- ETL、自动化构建、批处理任务调度

## 四、交付物说明

### 1. 核心功能

- `core/`：基础状态迁移引擎、守卫条件、动作回调、生命周期钩子
- `hsm/`：分层状态结构、事件冒泡、LCA 路径处理
- `workflow/`：顺序工作流、步骤状态、失败重试
- `timer/`：确定性定时事件与周期调度
- `export/`：导出 Mermaid 与 PlantUML 图

### 2. 使用方式

项目以 MoonBit 包形式组织，用户可以通过各模块 API 直接构建状态机、编排任务流或生成图表，并在示例目录中找到可运行参考。

### 3. 文档与说明

- `README.md`：项目总览、结构说明、质量检查说明
- `README.mbt.md`：可执行文档，保证示例代码可编译
- 示例目录：`examples/ai_agent_workflow/`、`examples/game_fsm/`

### 4. 测试规划

- 单元测试覆盖核心迁移逻辑、HSM 冒泡逻辑、工作流重试逻辑、定时器逻辑和导出逻辑
- CI 中固定执行 `moon check`、`moon fmt --deny-warn`、`moon info --deny-warn`、`moon test`

## 五、技术路线

### 1. 系统架构

- 底层 `core` 提供统一状态迁移抽象
- `hsm` 在 `core` 之上增加层级树与冒泡分派
- `workflow` 和 `timer` 作为独立能力模块并行存在
- `export` 从运行时对象生成结构化图形输出

### 2. 设计原则

- Contract-first：先定义接口与契约，再实现模块
- 可测试：优先使用确定性逻辑，避免依赖真实时间
- 可扩展：模块边界清晰，后续可继续增加并行工作流、条件分支和持久化能力
- 可复现：所有示例与测试都可在 MoonBit 工具链下重复验证

### 3. MoonBit 工具链

项目面向最新 MoonBit 工具链使用习惯进行整理，CI 计划固定使用 `0.10.3` 版本，确保格式、信息检查和测试结果一致。

## 六、风险分析与应对

### 1. 技术风险

- 复杂状态跳转容易出现生命周期调用顺序错误
- 工作流重试和失败恢复可能引入状态不一致
- 图导出与核心模型可能在后续演进时不同步

### 2. 应对措施

- 每个核心模块配套单元测试与示例测试
- 通过 CI 把格式、信息和测试统一纳入检查
- 让 README 与可执行文档保持同步，减少实现与说明分离
- 后续扩展前先明确模块职责和数据结构边界
